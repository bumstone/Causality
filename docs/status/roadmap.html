P2 스킬 절차 깊이

현재 distill이 event_type:kind 문자열 trace만 생성(skills.py). 재사용은 배선됐으나(P1) 내용이 얕다.
다음 재현 가능한 절차(도구·인자·아티팩트 바인딩)로 distill 강화 + 의미 기반 dedup(현재는 objective 소문자 매칭).
영역: skills · 효과: 재사용 실효성

P2 Verifier 실질 검증

현재 게이트가 distinct-verifier 2-pass는 집행(gates.py, 이름 기준 중복 방지). 그러나 검증의 내용/실질은 무검증.
다음 공허·형식적 verifier 통과 차단(검사 대상·증거 연결 요구) — 독립성을 이름 너머로.
영역: review/safety · 효과: 게이트 신뢰도

P2 Intent 분류 강건성

현재 키워드 휴리스틱(agent_harness.py re.search(\b…)). 키워드 없으면 TRIVIAL로 떨어져 governance 우회 가능.
다음 미분류·모호 시 보수적 게이트로 폴백(안전측), 또는 의미 분류 보강.
영역: execution/safety · 효과: 우회 방지

P3 End-to-end 실사용 검증

현재 단위·스모크 위주(test_loop.py). 다회 반복·다계약 전 경로(게이트→distill→승급→재사용) 미검증.
다음 5–10 반복 실워크로드 시나리오 테스트로 "루프 닫힘"을 실측.
영역: process · 효과: 닫힘의 증거

P3 실행 어댑터 + Playbook 실체화

현재 범용 work 어댑터는 browser_adapter뿐. 라우팅(_ROUTING)은 라벨만 부여, 플레이북 벤더링·실행 없음.
다음 subprocess/file/API 어댑터 번들 + 플레이북을 참조→실행/검증 가능하게.
영역: execution/memory · 효과: 운영성

P3 autofix 워크플로 레드 제거

현재 ANTHROPIC_API_KEY 시크릿 미설정 → Codex 리뷰 PR마다 autofix 잡 ❌(기능은 의도상 inert). test 잡은 green.
다음 시크릿 부재 시 green-skip 가드(레드 노이즈 제거) + 활성화(App 설치+시크릿) 1회 안내.
영역: process · 효과: CI 신호 정합

후순위 원장 rotation/.idx + 토큰 예산

원장 파일 무한 증가 — R4f 캐시는 메모리 한정. 규모 시 rotation/archival + 영속 .idx(ADR 0011 §3).
토큰 contract+tail 항상 주입분의 상한·절단 메커니즘 부재.
영역: performance/token · 트리거: 규모
