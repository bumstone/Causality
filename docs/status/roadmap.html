🚧 남은 작업

P0-A 집행 게이트 배선

현재 engine.run_task의 step이 work→run_review만 실행. 실행 게이트 미강제.
다음 action 직전 can_execute_action·check_tool_allowed·check_non_goal를, 루프 진입 전 evaluate_plan을 통과시키는 ExecutionAdapter로 work를 감싼다. ESCALATE/STOP이면 중단.
의존: 없음(독립) · 예상: 중간 규모 · 가장 높은 우선순위

P0-B failures → non_goals 환류 + TTL 집행

현재 Reflect가 scoped failure를 record_failure로 기록만. bind는 호출자 non_goals만 사용.
다음 bind 전에 entries("failures", active_only=True)로 유효(TTL 미만료)·scope 일치 failure를 회수해 non_goals로 주입(HITL 확인). 이걸로 TTL 루프 집행도 완성.
의존: TTL 메커니즘 ✅(#15) · 예상: 소~중 규모

P1 earned skill 재사용

현재 SkillStore.promoted() 읽기만 가능. dispatch/context 자동 주입 없음.
다음 task type/objective로 promoted 스킬을 회수해 context에 주입. authored > earned 우선순위.
의존: P0 게이트·환류 권장 선행 · 예상: 소~중 규모

후순위 R4f 전체 read-path 인덱싱

현재 R2 캐시로 append는 O(1). events() 조회는 매번 디스크 재파싱.
다음 원장 규모가 커질 때만 오프셋(.idx) 인덱스 검토. 현 규모엔 과설계.
의존: 없음 · 트리거: 성능 필요 시

✅ 반영됨 (이미 진행)

TTL 메커니즘 #15

is_expired · entries(active_only) · revoke · sweep + scope·ttl 지원 record_failure.
가드레일이 영구 래칫이 되지 않게 — 만료/철회 가능. (루프 집행 연결은 P0-B)

R2 ledger 인덱싱 #11

size-guarded latest-hash 캐시로 append O(N²)→amortized O(1). events_for_contract/latest_hash_for_contract 접근자로 출처 오염 footgun 제거.
ADR 0011 §2.1

R4 durable write #13 #14 #17

공통 durable.py — rewrite=temp+fsync+os.replace(원자), append=fsync+torn tail 복구, flock writer 직렬화(체인 fork 방지).
ADR 0011 §2.2 · 손상복구·동시성 테스트 포함
