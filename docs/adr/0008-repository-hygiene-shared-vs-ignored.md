# ADR 0008 — 저장소 위생: 공유 설계 vs 무시할 런타임 state

- **상태(Status):** Accepted — 구현(`.gitignore`, 2026-06-09)
- **관련:** [ADR 0005](0005-identity-memory-skill-substrate.md) · [ADR 0007](0007-context-economy-progressive-disclosure.md)

## 1. 동기 (Context)

산출물이 늘면서 "무엇을 저장소에 공유(commit)하고, 무엇을 로컬에만 두나(ignore)"를
정할 필요가 생겼다. 특히:

- `memory.py`(ADR 0005)는 런타임에 `memory/<type>/log.jsonl`을 기록한다.
- 루프/아젠다(ADR 0005 §2.3, 0006 §6.1)는 개발 진행용 `state.json` 같은 파일을
  가질 수 있다.

기존 `.gitignore`에 이미 **`.ouroboros/`** 가 들어 있다 — 즉 *설치된 자동화 파일과
런타임 ledger는 공유하지 않는다* 는 선례가 이미 있다. 본 ADR은 그 선례를 일반 규칙으로
승격한다.

핵심 경계 질문(사용자 제기): 개발 상세 `state.json`은 공유하면 안 되고, 반대로 설계
내용을 담은 `memory.md`를 **함께 공유**해야 한다면 ignore를 풀어야 하나?

## 2. 결정 (Decision)

산출물을 3분류한다.

| 분류 | 예시 | 처리 |
|---|---|---|
| **공유 설계/소스** | `src/`, `tests/`, `docs/adr/`, `README*`, `examples/`, 부트스트랩 *템플릿/구조 코드*, 구조 인덱스 `memory/<type>/README.md`, CI 설정 | **commit** |
| **런타임/개발 state** | `.ouroboros/`(ledger·설치물), 머신 생성 메모리 로그 `memory/**/*.jsonl`, `state.json`/`*.state.json`, 캐시·빌드 | **ignore** |
| **경계(큐레이션 공유물)** | 사람이 작성해 공유하려는 `memory/**/*.md` (확정된 설계 결정 노트 등) | **commit (기본 추적)** |

규칙을 `.gitignore`로 표현:

```gitignore
state.json
*.state.json
memory/**/*.jsonl   # 머신 로그만 무시
# memory/**/*.md 는 무시하지 않음 → 큐레이션 노트는 해제 없이 바로 첨부 가능
```

### 2.1 두 가지 메모리 산출물을 구분 (사용자 질문의 핵심)

- **메모리 *구조/스키마*** = 아키텍처. 코드(`memory.py`)와 구조 인덱스
  (`memory/<type>/README.md`)로 표현 → **공유**.
- **메모리 *런타임 내용*** = 특정 프로젝트의 누적 결정/가정/실패 로그(`*.jsonl`).
  민감정보·노이즈·재현 불가 컨텍스트를 포함할 수 있음 → **무시**.
- **큐레이션 메모리 노트(`*.md`)** = 사람이 공유 의도로 작성한 설계 기록.
  포맷을 `.md`로 두면 위 무시 규칙(`*.jsonl`)에 걸리지 않으므로 **별도 해제 없이**
  커밋 가능. (즉 "memory.md를 첨부해야 하면 해제?" → 해제 불필요, 처음부터 추적됨.)

### 2.2 새 파일을 만들 때의 운영 규칙

- 머신이 매 실행마다 쓰는 상태 파일을 새로 만들면(예: 루프/아젠다 `state.json`)
  **먼저 `.gitignore`에 추가하고 작업**한다.
- 공유가 목적인 설계 산출물은 소스 또는 `.md`로 만든다(자동 추적).

## 3. 검토한 대안 (Alternatives)

| 대안 | 판단 |
|---|---|
| A. `memory/` 전체 무시 | **기각** — 공유용 큐레이션 `.md`까지 막혀 사용자 요구와 충돌 |
| B. 모든 산출물 커밋 | **기각** — 런타임 로그의 노이즈·민감정보·비결정성이 diff/이력 오염 |
| C. 케이스별 수동 판단 | **기각** — 비일관·실수 유발. 규칙으로 고정 |
| D. (채택) 3분류 + 포맷 기반 규칙(.jsonl 무시 / .md·소스 추적) | **채택** |

## 4. 영향 (Consequences)

**긍정:** PR diff가 설계·소스에 집중되어 깔끔. 런타임 노이즈·민감정보 유입 차단.
공유 설계는 소스/`.md`로 명시적으로만 들어옴.

**부정/비용:** 기여자가 "이건 공유물인가 런타임인가"를 의식해야 함(본 ADR이 기준 제공).
`memory/**/*.jsonl` glob은 git의 `**` 지원에 의존(현행 git 충족).

**중립:** 기존 `.ouroboros/` 무시와 일관. 타깃 프로젝트의 `.gitignore`는 부트스트랩이
별도로 다룰 수 있다(후속).
